Evaluation Refactoring & Expansion Plan
Current State

35 tests running full pipeline for each stage (planner runs for retrieval tests, planner+retrieval run for curation tests)
Expensive (~$0.64/run), slow, non-deterministic failures hard to debug
Only 2 tests use LLM-as-judge (negative constraints)

Goal State

Isolated stage testing with hardcoded inputs
~40-45 tests total with better coverage
Each stage independently testable
LLM-as-judge for semantic validation
Fast, cheap, deterministic where possible

Expected Outcomes

Before: ~$0.64/run (full pipeline for each test)
After: ~$0.45-0.50/run (isolated tests + LLM-as-judge)

Retrieval test fails → knows retrieval is broken (not planner)
Curation test fails → knows curation is broken (not retrieval)
Clear failure attribution

Better Coverage

Current: Tests coordination more than individual agent logic
New: Tests each agent's actual decision-making
New tests cover prose quality (not just structure)

Execution Time

Faster (fewer LLM calls per test)
More deterministic (hardcoded inputs)
Still keeps integration tests for coordination

Key Principles

Isolation: Each stage tests its own logic with controlled inputs
Shuffling: Mix constraint/wrong-genre books throughout, not at end
LLM-as-judge: Use for semantic validation (genre, prose), not keyword matching
Factories: Centralize test data creation, don't duplicate logic
Realism: Use real tool queries for candidates, not hardcoded book lists
Integration: Keep full pipeline tests for coordination testing
