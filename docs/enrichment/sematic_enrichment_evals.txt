Phase 2: Build Missing Indexes
2.1 Baseline-Clean Index
Script: app/semantic_index/build_baseline_clean_index.py
Data Source:
sqlSELECT 
    b.item_idx,
    b.title,
    a.name as author,
    GROUP_CONCAT(os.subject) as ol_subjects
FROM books b
LEFT JOIN authors a ON b.author_idx = a.author_idx
LEFT JOIN book_ol_subjects bos ON b.item_idx = bos.item_idx
LEFT JOIN ol_subjects os ON bos.ol_subject_idx = os.ol_subject_idx
GROUP BY b.item_idx
HAVING ol_subjects IS NOT NULL
Embedding Template:
pythontext = f"{title} — {author} | subjects: {ol_subjects}"
Output: models/data/baseline_clean/

semantic.faiss
semantic_ids.npy
semantic_meta.json

Key Differences from Baseline-Old:

No description field
Only books with OL subjects (skip books with no subjects)


2.2 V1-Subjects Index
Script: app/semantic_index/build_v1_subjects_index.py
Data Source:
sqlSELECT 
    b.item_idx,
    b.title,
    a.name as author,
    GROUP_CONCAT(bls.subject) as llm_subjects
FROM books b
LEFT JOIN authors a ON b.author_idx = a.author_idx
LEFT JOIN book_llm_subjects bls ON b.item_idx = bls.item_idx
WHERE bls.tags_version = 'v1'
GROUP BY b.item_idx
HAVING llm_subjects IS NOT NULL
Embedding Template:
pythontext = f"{title} — {author} | subjects: {llm_subjects}"
Output: models/data/enriched_v1_subjects/

### **3.3 Report Features**

**Visual Aids**:
- **Overlap highlighting**: Books appearing in multiple models get colored borders
  - Green: appears in 5-6 models (high consensus)
  - Yellow: appears in 3-4 models
  - Red: unique to 1-2 models (interesting divergence)
- **Position indicators**: Show if book moved up/down between models
- **Coverage badges**: Show how many books each index contains

**Research Helpers**:
- Click book title → Google search
- Click author → Author search
- Click item_idx → Open Library page
- "Research all unfamiliar books" button → opens tabs for unknowns

**Evaluation Tracking**:
- LocalStorage saves after each query
- Progress bar shows X/Y queries evaluated
- Export button → download JSON with all evaluations
- Import button → reload previous session

**Keyboard Shortcuts**:
- `n` → next query
- `p` → previous query
- `s` → save current evaluation
- `e` → export all evaluations
